## **Git Lab 2 ‚Äì Branching**

**Commands I learned**
- `git branch` ‚Üí create a new branch
- `git checkout` ‚Üí switch to a branch
- `git merge` ‚Üí combine changes from another branch
- `git push` ‚Üí upload branch to GitHub
- `git pull --allow-unrelated-histories` ‚Üí merge remote history with local

**My Understanding**
Branches are like parallel timelines of the project:
- **Master branch** ‚Üí the main timeline (final version)
- **Feature branch** ‚Üí a scratch timeline where I try new changes

I can work safely in a feature branch without disturbing master.  
When ready, I merge the feature branch back into master.  

It feels like having a clean notebook page (**master**) and a scratch page (**feature-1**).  
I doodle on the scratch page, and if I like it, I glue it back into the main notebook.
